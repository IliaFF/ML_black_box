# Итоговый отчёт: Dz5_ML

**Задача:** реверс-инжиниринг чёрного ящика `y = f(x)` по 13 физическим признакам через активное обучение + поиск формулы.  
**Результат соревнования:** **88.735** (финальный балл на лидерборде).

---

## 1. Постановка задачи

Чёрный ящик принимает 13 признаков:
- **Физические:** `muon_flux` (μ), `cerenkov_angle` (θ), `drift_bias` (db), `track_curvature` (tc)
- **Gate-признаки (9 шт.):** `scint_alpha`, `pixel_jitter`, `cathode_temp`, `anode_noise`, `beam_phase`, `shield_rho`, `veto_count`, `gain_offset`, `trigger_rate`

Тест — равномерная сетка в $(-9, 9)^{13}$.  
Квота API исчерпана 2026-05-10.

---

## 2. Хронология работы

### Этап 1 — Инфраструктура и сбор данных (01–04 мая)

- Conda-среда, pipeline `collect.py` с active learning (Байесовская линейная регрессия + UCB)
- Протокол: dry-run → smoke (5 pts) → production
- Собрано **44 604 строки** (`dataset_full.parquet`), из них **29 092 активных** (all |feat| < 9)
- EDA: Bokeh, Plotly 3D, SHAP scatter, режимный анализ по всем 9 gate-признакам

### Этап 2 — Поиск архитектуры формулы (01–07 мая)

32 раунда бенчмарков (`architecture_formula_round{1..32}_*.py`):

| Раунд | Что тестировалось | Вывод |
|-------|-------------------|-------|
| 1–3 | Базовые синусоидальные, BW-резонансные, сплайновые семейства | BW+sin лучший кандидат |
| 4–6 | Остаточные, маршрутизирующие, OOD-устойчивые архитектуры | Single-formula устойчивее ансамблей |
| 7–9 | Структурные зонды, pole-families, `tan`/`sinc`-мотивы | Подтверждение BW-полюса |
| 10–15 | Teacher-oracle, multitask, dual-target, сохранение синусоиды | Явная формула > latent модели |
| 16–18 | Факторизация, latent argument via ML | ML-аргумент синуса не помог |
| 19–22 | OOD-ensemble, disagreement-cap политики | Caps помогают OOD, не core-zone |
| 23–32 | Damped/piecewise, unified oscillation, legacy integration | Единая явная формула оптимальна |

**Ключевой вывод:** структура `F(μ)·sin(k₁θ) + H(μ) + g_θ(θ)·cos(k₂μ + tc²/10) + μ·tc·C(θ)` объясняет данные лучше любой ML-архитектуры.

### Этап 3 — Формула K4 (36 параметров, 07–08 мая)

```
y = (F(μ)·sin(k₁θ) + H(μ) + g_θ(θ)·cos(k₂μ + tc²/10) + μ·tc·C(θ)) · G_main
```

- **F(μ)** — чётный BW-резонанс, пик при |μ|≈9.7
- **H(μ)** — нечётный двойной BW, сдвиг фона
- **g_θ(θ)** — θ-модулятор косинуса (только θ<0 в K4)
- **C(θ)** — коэффициент tc-связи, sinh + гауссиан
- **G_main** — произведение gate-функций clip(10-|x|, 0, 1)

Оптимизация: Nelder-Mead (5 стартов × 50K итераций) → RMSE формулы = **8.996** на active zone.

### Этап 4 — LGBM на резидуалах (08–10 мая)

```
y_pred = formula(μ, θ, tc, G) + LGBM(μ, θ, tc)
```

Прогресс ансамбля:

| Раунд | Строк active | RMSE (test 20%) |
|-------|-------------|-----------------|
| Baseline (формула K4) | 14 618 | 8.671 |
| Первый LGBM | ~18 848 | 3.33 |
| Round 4 | ~26 768 | 3.053 |
| Round 6 | ~27 768 | 2.966 |
| **Round 7** | **29 092** | **2.750** |

LGBM конфиг финала: 127 листьев, reg_lambda=20, min_child=50.

### Этап 5 — Формула V2 (44 параметра, 10–11 мая)

Расширение K4:
- **g_θ(θ≥0) ≠ 0**: добавлен гауссиан для положительных θ (в K4 был ноль)
- **C_neg**: +2 гауссиана
- **C_pos**: +3 гауссиана (был 1)
- 3 параметра вырождены: α_F≈5×10⁻⁵, β_tc≈10, A_T⁺≈0

### Этап 6 — 5-fold LGBM CV + финальная сабмиссия (11–13 мая)

```
OOF RMSE (active, 29K строк): 2.5464
Kaggle score: 88.735
```

Схема:
```python
y_pred = formula_V2(mu, theta, tc, G) + mean([lgbm_k(mu, theta, tc) for k in 5])
```

- Гиперпараметр поиск: 11 конфигов × 5-fold
- Финал: 5 моделей, сохранены в `lgbm_cv_fold{0..4}.txt`

## 3. Ключевые открытия

### Структура функции

| Компонента | Физический смысл |
|------------|-----------------|
| F(μ)·sin(k₁θ) | Основной BW-резонанс, амплитуда ~3200 при |μ|≈9.7 |
| H(μ) | Нечётный фоновый сдвиг по μ |
| g_θ·cos(k₂μ + tc²/10) | Фазовая модуляция, существенна при θ<0 |
| μ·tc·C(θ) | Линейная связь μ и tc с θ-зависимым коэффициентом |
| G_main | Все gate-признаки — чистые пороговые ворота (threshold ≈ ±9) |

### Что не сработало

| Идея | Почему не помогло |
|------|-------------------|
| ML-аргумент синуса (round18) | Линейная проекция не даёт новой геометрии аргумента |
| OOD-ансамбли (round19-22) | Помогают только за пределами active zone |
| tc² в LGBM (первый запуск) | abs_tc=0.2%, tc²=1.4% — незначимы. tc³ лучше |
| Нейросети | Исключены намеренно (нет GPU в данном проекте) |

### Худшие зоны (ошибка формулы+LGBM)

| Зона θ × |tc| | n | RMSE |
|-----------|---|------|
| [+6,+7) × [7,9) | 305 | 4.85 |
| [0,+1) × [7,9) | ~200 | 4.59 |

Причина: при больших |tc| и |μ|≈9 формула даёт экстремальные значения (F(μ)≈3200), LGBM не успевает скорректировать.

## 4. Финальная архитектура

```python
# Шаг 1: формульное предсказание
y_formula = predict_V2(mu, theta, tc, G_main, p_v2)  # 44 params

# Шаг 2: 5 LGBM-моделей на резидуалах (features: mu, theta, tc)
y_lgbm = mean([model_k.predict([mu, theta, tc]) for k in range(5)])

# Итог
y_pred = y_formula + y_lgbm  # active zone only (all |feat| < 9)
# inactive: skip (NaN → dropna в submission)
```
